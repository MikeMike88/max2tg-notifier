# Нотификатор MAX → Telegram

MAX убрали из App Store, push-уведомления на iPhone не приходят. Этот скрипт
слушает твой аккаунт MAX на ПК или сервере и пересылает новые сообщения в
Telegram — с **текстом** сообщения (для медиа — пометка вроде «📷 Фото») и
**только по незамьюченным чатам**. При желании можно слать вместо текста лишь
короткий **сигнал** «🔔 новое сообщение в таком-то чате», чтобы сам текст
оставался только в MAX на телефоне (`INCLUDE_MESSAGE_TEXT = False`, см.
настройку).

Работает на **Windows, Linux и macOS**. Под капотом —
[PyMax](https://github.com/MaxApiTeam/PyMax) (userbot-обёртка над внутренним API
MAX) и Telegram Bot API.

> ⚠️ PyMax использует **неофициальный внутренний API** MAX: это нарушает условия
> сервиса и теоретически возможен бан аккаунта (привязан к твоему номеру). Для
> личного использования риск низкий, но не нулевой.

## Установка

Нужен Python 3.10+ (проверено на 3.12).

```bash
python -m pip install -r requirements.txt
```

На Linux/macOS, если запускаешь headless (без значка в трее), `pystray` и
`Pillow` не нужны — можно поставить только ядро:

```bash
python -m pip install "maxapi-python>=2.3.1" "aiohttp>=3.13"
```

## Настройка

1. Создай телеграм-бота: **@BotFather** → `/newbot` → скопируй **токен**.
2. Скопируй шаблон конфига и впиши свои значения:
   ```bash
   cp config.py.example config.py      # Linux/macOS
   copy config.py.example config.py    # Windows
   ```
   В `config.py` укажи `MAX_PHONE` и `TELEGRAM_BOT_TOKEN`.
3. Узнай `chat_id`: напиши боту любое сообщение, затем
   `python get_telegram_chat_id.py` → впиши результат в `TELEGRAM_CHAT_ID`.

`config.py` и папка `cache/` в `.gitignore` — твои секреты и сессия в
репозиторий не попадают.

## Первый запуск и авторизация

Авторизация встроена в `main.py`. При первом запуске MAX пришлёт **SMS-код**, а
если на аккаунте включён **2FA — спросит пароль**; и то и другое вводится прямо
в консоли:

```bash
python main.py
```

Сессия сохранится в `cache/main.db`, дальше вход не потребуется. Чтобы войти
заново (сменился номер, протухла сессия) — `python main.py --login`.

После входа в Telegram придёт «✅ Нотификатор запущен». Напиши себе в
незамьюченный чат MAX — прилетит «🔔 Новое сообщение в …». `chat_id` каждого
чата виден в логе (`cache/notifier.log`).

## Запуск в фоне

### Windows (значок в трее)

После первого входа (см. выше) фоновый запуск со значком в трее:

```powershell
python main.py
```

Когда сессия уже есть, на Windows автоматически поднимается значок в трее
(меню «Открыть лог» / «Выход»). Для автозапуска при входе в систему положи
ярлык на `start_hidden.vbs` (запуск без мелькания окна) в `shell:startup`.
`pythonw` должен быть в `PATH`.

### Linux / macOS (headless + systemd)

На сервере без графики запускай в headless-режиме:

```bash
python main.py --no-tray
```

Пример systemd-юнита — `deploy/max2tg.service` (поправь `WorkingDirectory` и
путь к Python). Сначала **один раз** авторизуйся вручную (`python main.py
--login`), чтобы появилась сессия, потом:

```bash
sudo cp deploy/max2tg.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now max2tg
```

## Mute (не беспокоить) — автоматически

Заглушённые в MAX чаты определяются **автоматически**, настраивать ничего не
надо. Признак приходит в LOGIN-ответе: `config.chats.{chatId}.dontDisturbUntil`
(`-1` — навсегда, `0` — нет, `>0` — до момента в мс). `main.py` читает это при
запуске и не шлёт сигналы по заглушённым чатам.

- Включено флагом `RESPECT_MAX_MUTE = True` в `config.py` (поставь `False`,
  чтобы получать сигналы даже из заглушённых).
- mute читается **один раз при старте**. Заглушил/разглушил чат в MAX →
  перезапусти нотификатор, чтобы он подхватил изменение.
- Дополнительно можно вручную глушить чаты через `IGNORED_CHAT_IDS`.

## Тонкая настройка (`config.py`)

- `COOLDOWN_SECONDS` — антиспам: не чаще одного сигнала на чат за N секунд.
- `SKIP_SERVICE_MESSAGES` — не пинговать по системным событиям чата («X теперь
  в MAX», «X вышел из чата» и т.п.); по умолчанию `True`.
- `INCLUDE_MESSAGE_TEXT` — присылать вместе с сигналом сам текст сообщения (для
  медиа — пометку вида «📷 Фото»); по умолчанию `True`. Учти: текст уходит через
  твоего бота на серверы Telegram, и антиспам `COOLDOWN_SECONDS` не применяется
  (приходит каждое сообщение). Поставь `False`, чтобы слать только сигнал без
  текста.
- `MESSAGE_TEXT_LIMIT` — макс. длина пересылаемого текста (символов), `0` — без
  обрезки; действует только при `INCLUDE_MESSAGE_TEXT = True`.
- `RESPECT_MAX_MUTE` — учитывать заглушённые в MAX чаты (по умолчанию `True`).
- `IGNORED_CHAT_IDS` — ручной игнор (страховка).
- `WATCHED_CHAT_IDS` — если непусто, следим **только** за этими чатами.
- `CHAT_NAMES` — человеческие названия для красивых сигналов.

## Файлы

- `main.py` — нотификатор (включает авторизацию при первом запуске).
- `config.py.example` — шаблон настроек (копируется в `config.py`).
- `get_telegram_chat_id.py` — помощник для chat_id.
- `start_hidden.vbs` — фоновый запуск на Windows без мелькания окна (значок в трее).
- `deploy/max2tg.service` — пример systemd-юнита для Linux.
- `cache/` — сессия MAX и логи (**доступ к аккаунту, никому не показывай**).

## Если перестало работать

MAX обновился → обнови обёртку: `pip install -U maxapi-python`.
